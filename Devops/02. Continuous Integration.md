## code:
the code is written based on the design notes and the feedback given by the testing phase

## fetch:
The code is then fetched from the repo and sent to the build phase

## build:
The code is then built as the software or artifact and then sent to the next phase

## test:
The code is then tested and evaluated.

## notify:
If any abnormality is detected in the code then a notification is sent to the developer

## feedback:
A feedback is then generated and then given to the developer


# Goal
The goal of CI is to detect the problem early on and stop it from multiplying
